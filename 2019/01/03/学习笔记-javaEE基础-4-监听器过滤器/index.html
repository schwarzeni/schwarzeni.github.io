技术

来源于从网页搭建入门Java Web

这里是过滤器(filter)和监听器(listener)专场

  • 过滤器(filter)
  • 监听器(listener)
  • 综合案例：登录控制

过滤器(filter)

作用：内容过滤处理，编码处理，权限处理

两种配置方式：web.xml @WebFilter

第一种

注意两点：过滤器串的顺序和配置文件中<filter-mapping>声明的顺序相同，其次，设置的<dispatcher>告诉该过滤器作用于那种类型的url访问，有以下几种，基础好的人应该都不陌生

  • FORWARD
  • ERROR
  • REQUEST
  • INCLUDE

而 @WebFilter 方式声明过滤器无法控制顺序，它还有一些参数，这里就不再列举

监听器(listener)

作用：统计在线人数，页面访问统计，应用启动时完成初始化工作，与Spring结合

分类

  • request类
    • ServletRequestListener（创建销毁）
    • ServletRequestAttributeListener (添加替换删除)
  • session类
    • HttpSessionListener（创建销毁）
    • HttpSessionAttributeListener(添加替换删除)
    • HttpSessionBindingListener(绑定解绑)
  • servlet类
    • ServletContextListener（创建销毁）
    • ServletContextAttributeListener(添加替换删除)
